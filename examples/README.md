# TalkDB examples

Four Jupyter notebooks, one per headline capability. Executed outputs are committed — you can read them on GitHub without running anything. To re-run, set `OPENAI_API_KEY` (or `ANTHROPIC_API_KEY`) in `.env` at the project root and open any notebook.

| # | Notebook | Shows |
|---|----------|-------|
| 1 | [01_quickstart.ipynb](01_quickstart.ipynb) | Seed a demo ecommerce DB, build the hybrid retrieval index, ask a simple `COUNT(*)`, then a three-table aggregation. |
| 2 | [02_conversation.ipynb](02_conversation.ipynb) | Multi-turn: "revenue by month" → "just Q4" → "by payment method". The session log shows how each follow-up was rewritten into a standalone question before SQL generation. |
| 3 | [03_insight_agent.ipynb](03_insight_agent.ipynb) | `with_insights=True`: pandas analyzer detects a 134% revenue trend, the narrator writes 2-4 sentences using only the analyzer's numbers (never hallucinated), and a line chart is auto-generated. |
| 4 | [04_community_registry.ipynb](04_community_registry.ipynb) | Install the bundled `stripe-semantic` package, list / search / inspect it, then uninstall. Shows the package format (YAML-only, no code) and how metrics + proven queries get loaded into the retriever. |

The notebooks use the **library API** (`Engine(get_settings())`) — not the CLI — so you can lift code directly into your own apps. Each notebook starts with a setup cell that locates the project root, loads `.env`, and fails fast if no LLM key is present.

## Regenerating

The notebooks are generated by [scripts/build_example_notebooks.py](../scripts/build_example_notebooks.py). To rebuild after API changes:

```bash
python scripts/build_example_notebooks.py
for nb in examples/*.ipynb; do
  jupyter nbconvert --to notebook --execute --inplace --ExecutePreprocessor.timeout=180 "$nb"
done
```
